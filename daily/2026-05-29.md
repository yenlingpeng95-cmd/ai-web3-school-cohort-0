## 📝 學習記錄

### 影片：AI Agents Explained Simply（by Tina Huang）

> 從零技術背景出發，用三層階梯看懂 AI Agent 到底是什麼。

#### 🪜 Level 1：LLM（大語言模型）
- ChatGPT / Gemini / Claude 都是「應用」，底層是 LLM
- 使用方式：人類給輸入 → LLM 產出輸出
- 兩個關鍵限制：
  - **知識有限**：沒有內部公司資料或用戶個人資訊
  - **被動回應**：等我們下指令才會動作

#### 🪜 Level 2：AI Workflow（AI 工作流）
- 人類預先定義好「路徑」（控制邏輯），LLM 照著走
- 範例：問「我的咖啡約會何時？」→ LLM 先去 Google Calendar 查 → 再回答
- **RAG（檢索增強生成）** 就是一種 AI Workflow — 讓模型回答前先去查外部資料
- 關鍵特徵：**人類是決策者**，LLM 只是執行者
- 即使有上百個步驟，只要路徑是人定的，就還是 Workflow，不是 Agent

#### 🪜 Level 3：AI Agent（AI 代理）
- **唯一的質變**：人類決策者被 LLM 取代
- 三項核心能力：
  1. **Reason（推理）** — 思考最佳做法（「要不要先編譯連結再提取資料？」）
  2. **Act（行動）** — 透過工具執行（用 Google Sheets vs Excel？）
  3. **Iterate（迭代）** — 自主評估並改進（叫另一個 LLM 檢查輸出品質，循環直到達標）

#### 🔄 ReAct 框架
- 就是 **Re**ason + **Act** = ReAct
- AI Agent 最常見的架構

#### 🧠 一句話記住
> **Workflow 是人畫路給 AI 走；Agent 是 AI 自己決定怎麼走，還會回頭調整。**

#### 真實案例
1. **Make.com Workflow 範例**：Google Sheets + Perplexity + Claude → 自動產 LinkedIn/IG 貼文。但輸出不滿意時，人得手動改 prompt → 這就是「沒有迭代能力」
2. **Andrew Ng 的 AI Vision Agent 示範**：輸入關鍵字 "skier"，Agent 自己推理滑雪者長怎樣、掃描影片片段、標記、回傳結果

### 💡 我的連結思考

今天的影片恰好是昨天預覽的 Week 2 內容（RAG → Agent → Frameworks）的絕佳入門！影片把 RAG 定位成 Level 2 工作流的一種，Agent 則是 Level 3。這個階梯式理解非常適合非技術背景的人建立 mental model。

和 Handbook 第三章（Context）的關聯：Agent 要有效「推理」的前提，就是 Context 要管理好 — 指令、任務、事實、知識、記憶五層都得顧到，Agent 才不會亂走。
