## 📝 學習記錄

### Web3 基礎密碼學

> Web3 的安全基石建立在密碼學之上。今天的學習從最核心的三個密碼學工具出發：**哈希函數、非對稱加密、數位簽章**。

---

#### 🔑 1. 哈希函數（Hash Function）

- **定義**：將任意長度的輸入，轉成固定長度的輸出（如 SHA-256 → 256 bits）
- **三大特性**：
  - **單向性**：從 hash 無法反推出原始資料
  - **抗碰撞性**：找不到兩個不同的輸入產生相同的 hash
  - **雪崩效應**：輸入改一點點，輸出就完全改變
- **Web3 應用**：
  - 區塊鏈區塊連結（每個區塊包含上一個區塊的 hash）
  - 交易 ID（tx hash）
  - 地址生成（公鑰 → hash → 地址）
  - Merkle Tree — 快速驗證大量交易是否被篡改

> 💡 **一句話記住**：Hash = 資料的指紋 — 輕量、唯一、不可逆。

---

#### 🗝️ 2. 非對稱加密（Asymmetric Cryptography）

- **核心概念**：一對密鑰 — **公鑰（public key）** 公開分享、**私鑰（private key）** 絕對保密
- **兩種用途**：
  - **加密**：用公鑰加密 → 只有私鑰持有者能解密
  - **簽章**：用私鑰簽署 → 用公鑰驗證簽署者身份
- **Web3 最常見的曲線**：**橢圓曲線密碼學（ECC）**，特別是 **secp256k1**（比特幣/以太坊）
- **為什麼用 ECC 而不是 RSA？**
  - ECC 在同等安全強度下，密鑰更短、計算更快
  - 對區塊鏈這種資源敏感的環境很關鍵

> 💡 **一句話記住**：公鑰像地址（給別人寄東西用），私鑰像鑰匙（只有你能開門）。

---

#### ✍️ 3. 數位簽章（Digital Signature）

- **流程**：
  1. 發送方用私鑰對交易/訊息簽名
  2. 接收方用發送方的公鑰驗證簽名
- **三大保證**：
  - **身份認證**（Authentication）— 確認簽署者是誰
  - **不可否認性**（Non-repudiation）— 簽了就不能賴帳
  - **完整性**（Integrity）— 簽名後資料被改就會驗證失敗
- **Web3 應用**：
  - 發送交易（錢包用私鑰簽署每一筆交易）
  - 登入簽章（SIWE / EIP-4361 — Sign-In with Ethereum）
  - 智能合約多籤錢包（需要多個簽名才能執行）

> 💡 **一句話記住**：簽章 = 蓋手印 + 封印 — 證明是你，而且沒人動過。

---

#### 🔗 4. Web3 中密碼學的實際串聯

以一條以太坊交易為例：

```
你發起一筆轉帳
  → 錢包用私鑰對交易簽章（證明是你）
  → 交易廣播到網路
  → 礦工驗證簽章（用公鑰確認身份）
  → 交易被打包進區塊
  → 區塊 header 包含上一區塊 hash（形成鏈）
  → Merkle Root 包含所有交易 hash（證明交易沒被改）
```

整個流程只用到了 **三種密碼學工具**，但它們交織成整套信任機制。

---

#### 🧠 核心總結

| 工具 | 類比 | 保證什麼 |
|------|------|---------|
| 哈希函數 | 資料指紋 | 資料完整性、不可篡改 |
| 非對稱加密 | 信箱 + 鑰匙 | 資料機密性、權限控制 |
| 數位簽章 | 蓋章 + 封條 | 身份認證、不可否認性 |

三者的共同點：**不依賴信任第三方，只依賴數學**。這就是 Web3「信任最小化」（Trust Minimization）的底層邏輯。

---

#### 📚 延伸興趣

- ECDSA vs EdDSA — 不同的橢圓曲線簽章算法，後者在 Solana 等鏈上常見
- 什麼是 zero-knowledge proof（零知識證明）— 密碼學的下一個前沿
- BIP32 / BIP39（助記詞 + HD 錢包）— 實際錢包怎麼管理這麼多私鑰
